# twotalBarca Newsroom Backend

`newsroom/` is the repo-local editorial backend for the autonomous newsroom core.

Directory contracts:

- `config/workflow.json`
  Encodes allowed statuses, transitions, required approvals, and the deferred distribution contract pointer.
- `content/articles/*.json`
  Editorial article records with status, assignment ids, approval ids, publishing fields, and deferred distribution state.
- `content/dispatch/*.json`
  Weekly dispatch records with status, approval coverage, packaged issue contents, and deferred distribution state.
- `state/frontpage.json`
  Active front-page plan. This can override the homepage hero and selected rails after validation.
- `state/editorial-calendar.json`
  Lightweight calendar of scheduled desk work and publication beats.
- `assignments/*.json`
  Work orders with owner, editor, due date, and definition of done.
- `approvals/*.json`
  Approval requests and decisions. Published content must point to an approved record here.
- `sources/*.json`
  File-based source registry entries that the ingest cycle turns into deterministic signals.
- `drafts/*.json`
  Canonical draft artifacts generated from newsroom assignments and bound back to editorial records.
- `reviews/*.json`
  Change requests, approval requests, and revision artifacts linked to drafts and records.
- `generated/signals/*.json`
  Deterministic trigger outputs created by the ingest cycle.
- `generated/publish-queue.json`
  Local publish queue derived from scheduled records and pre-publish diagnostics.
- `generated/review-summary.json`
  Reviewer-facing summary of pending approvals and blocked records.
- `generated/site-content.json`
  Deterministic compiled payload consumed by the public site. Generated by `npm run newsroom:build`.

Operator commands:

- `npm run newsroom:bootstrap`
- `npm run newsroom:validate`
- `npm run newsroom:build`
- `npm run newsroom:dashboard`
- `npm run newsroom:ingest`
- `npm run newsroom:trigger-cycle`
- `npm run newsroom:generate-draft -- --assignment assignment-montjuic-culture-column`
- `npm run newsroom:request-approval -- --kind article --slug montjuic-needs-a-louder-start`
- `npm run newsroom:publish-cycle`
- `npm run newsroom:healthcheck`
- `npm run newsroom:workflow -- --kind article --slug rest-defense-is-learning-to-breathe --to approved`
- `npm run newsroom:assignment -- --title "New desk assignment" --owner editor-in-chief --approver editor-in-chief --kind article --deadline 2026-04-07`

Publishing rules:

- `draft -> assigned -> in_progress -> in_review -> copy_review -> approved -> scheduled -> published`
- a content item cannot move into `approved`, `scheduled`, or `published` without the required approval roles from `newsroom/approvals/`
- homepage overrides only apply when `newsroom/state/frontpage.json` resolves to published newsroom content; otherwise the public site falls back to seeded content
- outbound distribution is intentionally deferred; see `docs/distribution-spec.md`
