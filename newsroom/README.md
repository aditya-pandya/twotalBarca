# totalBarca Newsroom Backend

Last updated: 2026-05-02

`newsroom/` is the repo-local editorial backend that supports the weekly totalBarca dispatch and its minimal match-context capsules.

Scope note:
- active output is the weekly dispatch issue
- active support context is one last-match capsule and one next-match capsule
- legacy article-first machinery may still exist here, but it is dormant/legacy unless explicitly reintroduced

## Directory contracts

- `config/workflow.json`
  Allowed statuses, transitions, required approvals, and the deferred distribution boundary.
- `content/dispatch/*.json`
  Active weekly dispatch records with issue structure, approvals, publish state, and distribution state.
- `content/articles/*.json`
  Legacy/dormant article records. Keep only as migration/history support unless the product scope changes.
- `state/frontpage.json`
  Active home/latest-issue plan. Should bias toward current dispatch issue and minimal match context.
- `state/editorial-calendar.json`
  Weekly planning notes and publication beat tracking.
- `assignments/*.json`
  Work orders for weekly issue assembly, supporting reporting, or platform cleanup.
- `approvals/*.json`
  Approval requests and decisions. Published dispatches must point to approved records here.
- `sources/*.json`
  Desk-only/internal source notes. Useful as support material, not as public product requirements.
- `generated/barca-live-source.json`
  Optional live scout artifact built from external context.
- `generated/barca-scout-report.json`
  Human-readable ranked input for choosing the week's five topics and relevant match context.
- `drafts/*.json`
  Canonical draft artifacts generated from assignments.
- `reviews/*.json`
  Revision and approval artifacts linked to drafts and records.
- `generated/signals/*.json`
  Deterministic trigger outputs created by ingest/scouting cycles.
- `generated/site-content.json`
  Deterministic compiled payload consumed by the public site. Generated by `npm run newsroom:build`.

## Operator commands

- `npm run newsroom:bootstrap`
- `npm run newsroom:validate`
- `npm run newsroom:build`
- `npm run newsroom:dashboard`
- `npm run newsroom:scout-barca`
- `npm run newsroom:ingest`
- `npm run newsroom:trigger-cycle`
- `npm run newsroom:generate-draft -- --assignment assignment-weekly-dispatch-14`
- `npm run newsroom:request-approval -- --kind dispatch --slug weekly-dispatch-issue-18`
- `npm run newsroom:workflow -- --kind dispatch --slug weekly-dispatch-issue-18 --to approved`
- `npm run newsroom:assignment -- --title "Weekly Dispatch No. 18" --owner editor-in-chief --approver editor-in-chief --kind dispatch --deadline 2026-05-08`

Practical note:
- when draft generation lookup is ambiguous, use the assignment id rather than the record slug

## Publishing rules

- `draft -> assigned -> in_progress -> in_review -> copy_review -> approved -> scheduled -> published`
- a record cannot move into `approved`, `scheduled`, or `published` without the required approval roles from `newsroom/approvals/`
- homepage overrides only apply when `newsroom/state/frontpage.json` resolves to valid published content; otherwise the public site falls back to seeded content
- outbound distribution is intentionally deferred; see `docs/distribution-spec.md`
- active editorial intent should center on one weekly issue with exactly five topics plus minimal match capsules
