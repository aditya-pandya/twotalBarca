# totalBarca Product Brief

Last updated: 2026-05-02

## 1. What this is

totalBarca is a weekly FC Barcelona dispatch product.

The active public offer is simple:
- one weekly issue
- exactly five sharp, opinionated topics
- one minimal last-match capsule
- one minimal next-match capsule

It should feel:
- calm
- edited
- opinionated
- football-native
- phone-friendly

It should not feel like:
- a rumor blog
- a feed
- a match center
- a generic sports dashboard
- a broad digital magazine with too many shelves

## 2. Product framing

This is no longer an article-first publication.
This is no longer a standalone Brief plus Dispatch product.

The useful idea from the old Brief survives inside the weekly Dispatch:
- exactly five commentary points
- each point has a take
- each point explains why it matters

## 3. Audience

Primary audience:
- Barça supporters who want cleaner judgment than the daily churn provides
- readers who value opinion with evidence instead of volume

Secondary audience:
- readers who care about club identity, tactics, governance, Femení, and La Masia when those angles materially matter that week

## 4. Active product contract

### Weekly Dispatch
- cadence: once per week
- structure: exactly 5 items
- each item must have:
  - a clear editorial take
  - concise commentary/opinion
  - why-it-matters framing
- topics may cover first team, Femení, La Masia, tactics, club governance, culture, or related Barça themes only when current evidence supports the choice
- the issue is not a roundup of links and not disguised longform packaging

### Match context
- last-match capsule: opponent, competition, date, result when known, one short read
- next-match capsule: opponent, competition, kickoff/date, venue if useful, one thing to watch
- keep both capsules minimal

## 5. Active public surfaces

1. Home / latest issue surface
2. Dispatch issue page
3. Small dispatch archive if needed
4. Tiny About/positioning blurb if needed

Match context is a module inside those surfaces, not a separate product surface.

## 6. Dormant or future-only surfaces

These are not active requirements right now:
- standalone Brief page
- article page as the product centerpiece
- analysis page
- culture page
- archive/vault shelf as a public promise
- match notes page
- topic/person/season routes
- reactions and community widgets

If they remain in code or old docs, treat them as dormant/legacy/background.

## 7. Public-copy rules

- public brand is `totalBarca`
- keep the locked tagline/supporting line
- no AI/process/status language on the public site
- never call it a blog
- do not present internal newsroom mechanics as public product copy

## 8. Success criteria

The site should immediately communicate:
- this is for Barça supporters
- this updates once a week, not every hour
- the five-topic dispatch is the main event
- match context is useful but restrained
- the whole thing is cleaner and calmer than fan-site churn
